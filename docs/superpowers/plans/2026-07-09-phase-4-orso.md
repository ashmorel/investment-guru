# Phase 4 — ORSO Environment — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the ORSO pension environment per spec `docs/superpowers/specs/2026-07-09-phase-4-orso-design.md` — fund menu, allocation snapshot + switch log, HKD pricing (HSBC fetcher attempt + manual fallback), retirement goals + deterministic projection, and the Guru's ORSO switching-advice mode.

**Architecture:** New `app/services/orso/` package (prices, projection, context) mirroring `market_data`'s provider pattern; ORSO advice reuses the Phase 2b `GuruService`/`guru_reports`/locks/usage machinery with `kind="orso"` and a fund-code validity check; one new API router `/api/orso/*`; one new frontend page. ORSO never touches the daily digest/take.

**Tech Stack:** FastAPI + SQLAlchemy 2 async + Alembic, httpx (fetcher), existing `anthropic` LLM layer, React/Vite/TS/Tailwind v4 + React Query.

## Global Constraints

- Public repo: **never commit real holdings data** — the seeded fund menu (public scheme data: codes/names/asset classes/risk ratings) is allowed; personal units/splits/prices are NOT seeded.
- Money/units = `Numeric`/`Decimal`, never float. Every user-data table has `user_id`.
- DB change = hand-written chained Alembic migration (`alembic heads` → `0005`; new head `0006`).
- Async tests: module-level `pytestmark = pytest.mark.asyncio(loop_scope="session")` when all tests are async; shared conftest fixtures (`client`, `auth_client`, `guru_client`, `db_session`, `fake_llm`).
- Providers fixture-mocked in tests; endpoints degrade on provider failure, **never 500**.
- TDD. Verify: `ruff check . && pytest` (backend, from `backend/`, venv `backend/.venv`), `npm run check` (frontend). Frontend tests mock `globalThis.fetch` via `vi.spyOn`; vitest-axe available.
- Disclaimer string: `"The Guru is not regulated financial advice."` (import `DISCLAIMER` from persona).
- Validation values (spec §6): units ≥ 0; 0 ≤ contribution_pct ≤ 100; price > 0 → 422. Split sum ≠ 100 = flag, not error. Fund codes normalised upper. Archive fund with units ≠ 0 → 409 `fund_has_units`. Stale price threshold: `as_of` older than **7 days**.
- Projection rates exactly `(0.02, 0.05, 0.08)`, monthly compounding.
- Commits end `Co-Authored-By:` trailer per session convention. Update `.superpowers/sdd/progress.md` after each task.

**Execution notes:** Tasks 1–6 backend; Task 7 = **user gate** (Figma approval — stop and wait); Task 8 frontend; Task 9 docs + live smoke + **final whole-branch review on Opus**. Push seams: after Task 6 and after Task 8.
